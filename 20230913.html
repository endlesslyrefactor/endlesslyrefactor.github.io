Next I'm going to introduce the new ID to use in the TodoCollection, without it I thing I will struggle with ordering and filtering the collection

From past experience, I like to have UUIDs used as IDs within code. The ability to generate them without checking it is unique is very useful. And if the object needs to be saved somewhere that a UUID has issues (such as in a SQL database) then it is possibl,e to have multiple IDs for these contexts

Now, currently, there is no need for an ID to be rehydrated. So I think I can get away with the ID just having a create method which will always generate a random value

This is what I came up with:

      
        import { ValueObject } from "./value-object";

        interface IdProps {
            value: string
        }
          
        class TodoId extends ValueObject {
            get value () : string {
              return this.props.value
            }
            
            private constructor (props: IdProps) {
              super(props)
            }
          
            public static create () : TodoId {
                return new TodoId({ value: crypto.randomUUID() })
            }
        }

        export default TodoId;

I think that is simple enough to not need any unit tests. However, I decided to check something I was worried about, TypScript's structural equality. It seems like currently, a TodoName and a TodoId will be considered equal if they have the same value. This isn't the behaviour I want (although I'm perfectly willing to admit this made be my C# mindset)

Thankfully it looks like I can easily fix this by changing the props for each value type to contain the type name, e.g. todoId rather than value

Next I changed TodoCollection to use the ID for deletes and prevent items with duplicates being added

      
        add(todo: Todo) : void {
          if (this.#todos.find(x => x.id == todo.id))
              throw new ArgumentError(`A Todo with an id of ${todo.id.value} has already been added`)
  
          this.#todos.push(todo)
        }
    
        delete(id: TodoId) : void {
          const index = this.#todos.findIndex(x => x.id == id)
  
          if (index >= 0) {
              this.#todos.splice(index, 1)
          }
        }
